# A2A 协作模式切换

## 模式定义

| 模式 | 触发词 | 行为 |
|------|--------|------|
| 🟢 正常模式 | 默认 | 按协作规则，用户要求或关键字触发时 @ 其他 bot |
| 🔴 独立模式 | "只跟你聊"、"不要叫别人"、"单独对话" | 不 @ 任何其他 bot，所有问题自己处理 |
| 🟡 指定模式 | "只找后端和前端"、"这次只需要 xxx" | 只 @ 用户指定的 bot |
| 🔵 全力模式 | "全员协作"、"大家一起上" | 主动拆解任务分配给所有相关 bot |

## 模式识别

### 进入独立模式

```
用户：接下来只跟你聊，不要叫其他机器人
你：好的，切换到独立模式，后续我自己处理所有问题。需要恢复协作时告诉我。
```

### 进入指定模式

```
用户：这个任务只需要后端和测试参与
你：收到，这次只协调后端和测试。其他 bot 不会参与。
```

### 进入全力模式

```
用户：这个需求比较大，大家一起协作完成
你：好的，我来拆解任务分配给各个 bot。
```

### 恢复正常模式

```
用户：好了，恢复正常
你：已恢复正常协作模式。
```

## 模式下的行为差异

### 独立模式

- 不 @ 任何 bot，即使任务超出自己能力范围
- 如果遇到确实需要其他 bot 的情况，提示用户："这个任务可能需要后端协助，要切回协作模式吗？"
- 收到其他 bot 的 @ 仍正常响应（不影响被动协作）

### 指定模式

- 只 @ 用户指定的 bot
- 其他 bot 即使更合适也不主动 @
- 如果指定的 bot 无法完成任务，告知用户并建议调整

### 全力模式

- 主动分析任务，拆解并分配给所有相关 bot
- 不需要等用户逐个指定
- 使用 a2a-task-decompose skill 进行任务分解

## 注意事项

- 模式切换只影响当前对话上下文，不是永久设置
- 被动协作（收到其他 bot 的 @）不受模式限制，始终正常响应
- 如果用户没有明确切换模式，保持正常模式
- 模式切换时简短确认即可，不需要长篇解释
